## Контекст

Работа была сосредоточена на создании полной системы документации для проекта FEONT на основе проверенной системы из проекта FEMSQ, адаптированной под специфику работы с графом знаний. 

Проект FEONT - это исследовательский инструмент для моделирования и эволюции бизнес-процессов через граф знаний на основе RDF/SHACL технологий. ## Технические решения

### Ключевые решения

**Решение 1: Структура документации**
- **Проблема:** Необходима система документации для исследовательского проекта с графом знаний
- **Решение:** Адаптация проверенной системы из FEMSQ под специфику FEONT
- **Обоснование:** Система из FEMSQ проверена на практике, её адаптация быстрее создания новой системы
- **Результат:** Создана полная система с учетом специфики графа знаний

**Решение 2: Приоритетное использование Desktop Commander MCP**
- **Проблема:** Нужно определить приоритеты использования инструментов для работы с файлами
- **Решение:** Desktop Commander MCP используется приоритетно, терминал только при недоступности
- **Обоснование:** Desktop Commander более надежен для операций с файловой системой
- **Результат:** Правила зафиксированы в `.cursorrules`

**Решение 3: Расположение кода в папке `code/`**
- **Проблема:** Нужно определить, где будет храниться исходный код
- **Решение:** Создана папка `code/` рядом с папкой `docs/`
- **Обоснование:** Четкое разделение документации и кода, простота навигации
- **Результат:** Структура определена и задокументирована

**Решение 4: Расширения онтологии**
- **Проблема:** Нужно детально документировать онтологию без перегрузки основного файла
- **Решение:** Созданы расширения в `docs/project/extensions/ontology/`
- **Обоснование:** Разделение основной документации и деталей, гибкость расширения
- **Результат:** Созданы файлы структуры, префиксов и словаря

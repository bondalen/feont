# Docker - FEONT

Docker конфигурации для развёртывания FEONT.

## Важно

**Основное приложение развёртывается как JAR файл, а не в Docker контейнере.**

Данная папка содержит опциональные Docker конфигурации только для:
- Минимального контейнера данных (если требуется изоляция)
- Development окружения (опционально)

## Рекомендация

Для максимальной экономии ресурсов **НЕ рекомендуется** использовать контейнер для данных. Вместо этого храните TDB2 напрямую на хосте: `/home/user1/feont/data/tdb2/`

## Структура

```
docker/
├── docker-compose.yml      # Docker Compose конфигурация
└── README.md
```

## Использование (опционально)

Если всё же требуется контейнер для данных:

```bash
# Запуск контейнера данных
cd docker
docker-compose up -d feont-data

# Проверка статуса
docker-compose ps

# Остановка
docker-compose down
```

Контейнер использует `busybox:latest` (~5MB) и только монтирует volume с данными.
